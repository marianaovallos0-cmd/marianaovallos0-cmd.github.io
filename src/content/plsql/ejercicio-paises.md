## Contexto

Ejercicio guiado en clase. El profesor proporcionó los comandos base y trabajamos sobre una estructura de tablas `PAIS` y `EQUIPO` para practicar llaves foráneas, bloques `DECLARE`, procedimientos almacenados e inserción masiva de datos.

---

## 1. Llave Foránea entre EQUIPO y PAIS

```sql
ALTER TABLE EQUIPO
  ADD CONSTRAINT "EQUI_PA_FK"
  FOREIGN KEY ("ID_PAIS")
  REFERENCES PAIS ("ID_PAIS")
  ENABLE;
```

Esto garantiza integridad referencial: no se puede insertar un equipo con un `ID_PAIS` que no exista en la tabla `PAIS`.

---

## 2. Bloque DECLARE — País con más clubes

```sql
DECLARE
  vv_pais PAIS.NOMBRE_PAIS%TYPE;
BEGIN
  SELECT P.NOMBRE_PAIS INTO vv_pais
  FROM PAIS P
    INNER JOIN EQUIPO E ON P.ID_PAIS = E.ID_PAIS
  GROUP BY P.NOMBRE_PAIS
  ORDER BY 1 DESC
  FETCH FIRST 1 ROWS ONLY;

  DBMS_OUTPUT.PUT_LINE('País con más clubes: ' || vv_pais);
END;
```

Se usa `%TYPE` para que la variable `vv_pais` herede automáticamente el tipo de dato de la columna `NOMBRE_PAIS`, lo que hace el código más robusto frente a cambios de esquema.

---

## 3. Procedimiento — Actualizar nombre de país a minúsculas

```sql
CREATE OR REPLACE PROCEDURE sp_actualizaPais (pn_codigoPais IN NUMBER)
IS
BEGIN
  UPDATE PAIS
  SET NOMBRE_PAIS = LOWER(NOMBRE_PAIS)
  WHERE ID_PAIS = pn_codigoPais;

  COMMIT;

  DBMS_OUTPUT.PUT_LINE('Se actualizó el país: ' || TO_CHAR(pn_codigoPais));
END;
```

El procedimiento recibe el código del país, aplica `LOWER()` al nombre y confirma el cambio con `COMMIT`. Usar `TO_CHAR` en el `PUT_LINE` convierte el número a texto para la concatenación.

## Aprendizaje

Este ejercicio integra varios conceptos: la llave foránea garantiza integridad referencial entre tablas relacionadas; el bloque `DECLARE` con `%TYPE` muestra cómo consultar y procesar un solo resultado; el procedimiento almacenado encapsula lógica reutilizable; y el dataset masivo con `INSERT` ilustra cómo poblar tablas de catálogo.
